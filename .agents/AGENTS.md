# Project-Scoped Rules

## UI Research-First Rule
Before redesigning or rebuilding any dashboard page (student, admin, or public portfolio), always perform deep web research first:
1. Search Dribbble, Behance, and design blogs for the specific dashboard type.
2. Find color palettes, layout patterns, and component ideas from 2024-2025 designs.
3. Present a design brief / implementation plan for user approval before writing any code.
4. Never build from scratch without referencing at least 3-5 real design examples.

Goal: Avoid generic, cookie-cutter designs. Every UI must feel premium and distinctive.
